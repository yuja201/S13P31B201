# 기여 가이드

## 시작하기

- 저장소를 포크하고 `feature/<짧은-설명>` 브랜치를 만듭니다.
- 코드 주석과 커밋 메시지는 영어로 작성합니다.
- PR 전에 기존 테스트를 실행해 실패를 해결합니다.

## 코드 기준

- 언어별 포매터/린터를 사용하고 경고를 무시하지 않습니다.
- 작은 단위의 커밋과 명확한 메시지(`<scope>: <summary>`)를 선호합니다.
- 동작 변경 시 테스트를 추가하거나 업데이트합니다.

## PR 체크리스트

- 문제, 해결 방법, 수행한 테스트를 설명합니다.
- UI/UX 수정 시 스크린샷이나 로그를 첨부합니다.
- CI가 통과하는지 확인하고 리뷰 피드백을 신속히 반영합니다.

## 커뮤니케이션

- 리뷰에서 간결하고 존중하는 톤을 유지하며, 피드백의 이유를 설명합니다.
- 초안 단계는 Draft PR로 공유하고 블로커를 명확히 표시합니다.
- PR 본문에 트레이드오프와 알려진 한계를 기록합니다.
