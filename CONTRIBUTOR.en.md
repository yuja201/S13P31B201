# Contributor Guide

## Getting Started

- Fork the repo and create a branch `feature/<short-description>`.
- Use English for code comments and commit messages.
- Run the existing test suite before opening a pull request.

## Coding Standards

- Follow the language formatter and linter; do not ignore warnings without reason.
- Prefer small, focused commits with clear messages: `<scope>: <summary>`.
- Add or update tests with any behavior change.

## Pull Request Checklist

- Describe the problem, solution, and testing performed.
- Attach screenshots or logs when fixing UI/UX issues.
- Ensure CI passes and respond to review feedback promptly.

## Communication

- Keep reviews concise and respectful; explain the “why” behind feedback.
- Use draft PRs for early feedback and mark blockers clearly.
- Note trade-offs and known limitations in the PR description.
